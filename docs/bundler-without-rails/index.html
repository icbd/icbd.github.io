bundler without Rails

2021-05-27
  • bundler

bundle install 是再熟悉不过的命令了, 但脱离了 Rails 环境, 如何使用包管理工具呢?

  • Gem: https://github.com/rubygems/rubygems
  • Bundler: https://github.com/rubygems/rubygems/tree/master/bundler

Demo

#!/bin/zsh
mkdir demo
cd demo

bundle init
bundle add rake --version "~> 12.0"
bundle install

cat > main.rb <<EOF
require "bundler/setup"
require "rake"

p Rake::VERSION

EOF

ruby main.rb

我本地的开发环境已经安装了三个版本的 rake:

此时, Rake 的最新版本是 13.0.3 .

上面脚本最后输出为 "12.3.3", 也就是按照我们 Gemfile 中声明的规则, 使用了 12 的最新版本.

.
├── Gemfile
├── Gemfile.lock
└── main.rb

Gemfile

# frozen_string_literal: true

source "https://rubygems.org"

git_source(:github) {|repo_name| "https://github.com/#{repo_name}" }

# gem "rails"

gem "rake", "~> 12.0"

main.rb

require "bundler/setup"
require "rake"

p Rake::VERSION

require "bundler/setup" 做的事情是: 读取 Gemfile.lock, 把指定版本的 Gem 添加到 LOADPATH 中.

所以使用 Rake 前, 还是需要 require "rake".

如果想 require 所有 default 分组的包, 可以这样:

require "bundler/setup"
Bundler.require(:default)

p Rake::VERSION
